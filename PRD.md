## 1. 产品概述

### 1.1 产品定位

AutoSight是一款面向中小型汽配卖家的蓝海选品SaaS工具，致力于帮助卖家通过数据驱动的方式发现高增长、低竞争、供需缺口的蓝海品类机会。

**核心价值主张**: "卖铲子，而非淘金" - 我们提供的是选品工具和数据洞察，而非直接的销售服务。

### 1.2 产品愿景

成为汽配行业最具影响力的智能选品平台，让每一个中小卖家都能享受到大公司级别的数据分析和选品能力。

### 1.3 目标市场

| 维度 | 描述 |
|------|------|
| **市场规模** | 中国汽配电商市场年规模超过2000亿元 |
| **目标用户** | 中小型汽配卖家、电商从业者、工厂直销商 |
| **用户痛点** | 选品靠直觉、数据获取难、竞争分析缺失 |

## 2. 目标用户画像

### 2.1 核心用户画像

#### 用户A: 成长型卖家
- **特征**: 店铺月销售额10-50万，已有稳定品类
- **痛点**: 想拓展新品类但缺乏数据支撑
- **诉求**: 快速找到蓝海品类，降低试错成本
- **付费意愿**: 中等，愿意为高效工具付费

#### 用户B: 新入局卖家
- **特征**: 初创店铺或跨行业转型
- **痛点**: 完全不知道卖什么，无从下手
- **诉求**: 系统性的选品指导和市场分析
- **付费意愿**: 较低，需要免费试用期教育

#### 用户C: 效率型卖家
- **特征**: 多店铺运营，需要快速决策
- **痛点**: 数据分散，分析耗时
- **诉求**: 一站式数据平台，高效产出
- **付费意愿**: 高，追求投入产出比

### 2.2 用户场景

| 场景 | 用户行为 | 期望结果 |
|------|----------|----------|
| 日常选品 | 每天上班第一件事查看蓝海推荐 | 快速发现新机会 |
| 品类拓展 | 有新品类需求时搜索筛选 | 找到适合的蓝海品类 |
| 竞品分析 | 查看品类竞争格局 | 了解市场供需关系 |
| 月度复盘 | 导出月度数据分析报告 | 总结优化选品策略 |

## 4. 功能详细说明

### 4.1 供需看板（Dashboard）

#### 功能描述
展示市场整体供需状况，为用户提供宏观市场洞察。

#### 功能清单
| 功能点 | 描述 | 备注 |
|--------|------|------|
| 总需求指数 | 显示当前市场需求热度 | 实时更新 |
| 供给增速 | 展示近期待上架品类增速 | 周更新 |
| 蓝海机会数 | 当前供需缺口品类数量 | 日更新 |
| 竞争强度 | 市场整体竞争程度指数 | 周更新 |

#### 交互设计
- 数据卡片支持悬停查看详情
- 图表支持时间维度切换（7天/30天/90天/年度）
- 核心指标支持点击跳转详情

#### 边界条件
- 数据加载中显示骨架屏
- 网络异常显示重试按钮
- 数据为空显示空状态引导

---

### 4.2 蓝海选品（Blue Ocean）

#### 功能描述
核心选品功能，通过供需缺口模型筛选蓝海品类。

#### 选品模型

**蓝海指数计算公式**:
```
蓝海指数 = (需求增速 × 0.4) + (供需缺口 × 0.3) + ((100-CR5) × 0.3)
```

**评分标准**:
- 蓝海指数 > 30: 顶级蓝海
- 蓝海指数 20-30: 蓝海
- 蓝海指数 < 20: 红海

#### 功能清单
| 功能点 | 描述 | 会员限制 |
|--------|------|----------|
| 蓝海品类列表 | 按蓝海指数排序展示 | 免费5条/天 |
| 多维度筛选 | 按品类类型、供需缺口、CR5等筛选 | 高级筛选需Pro |
| 排序功能 | 支持按供需缺口、需求增速等多维度排序 | - |
| 品类详情 | 查看品类完整数据和分析建议 | - |
| 收藏功能 | 关注感兴趣的品类 | 需登录 |
| 今日推荐 | AI算法推荐的Top品类 | 需登录 |

#### 数据字段
| 字段 | 类型 | 描述 |
|------|------|------|
| 品类ID | String | 唯一标识 |
| 品类名称 | String | 品类标准名称 |
| 品类类型 | Enum | 新能源/定制件/宠物/电子/通用 |
| 供需缺口 | Float | 需求-供给的百分比差值 |
| 需求增速 | Float | 近30天需求增长率 |
| 供给增速 | Float | 近30天供给增长率 |
| CR5指数 | Float | 前5名商家占比 |
| 蓝海状态 | Enum | 顶级蓝海/蓝海/红海 |

#### 品类类型定义
| 类型 | 说明 | 示例 |
|------|------|------|
| 新能源配件 | 新能源汽车相关配件 | 充电桩、电池保护壳 |
| 电子配件 | 车载电子设备相关 | 行车记录仪、无线充电 |
| 定制配饰 | 可个性化定制的产品 | 氛围灯、内饰贴膜 |
| 宠物车载 | 宠物出行相关配件 | 宠物安全座椅、车载笼子 |
| 通用配件 | 无特定分类的通用产品 | 脚垫、座垫 |

## 6. 非功能性需求

### 6.1 性能指标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 页面加载时间 | < 3秒 | 首屏加载完成 |
| API响应时间 | < 500ms | 核心接口 |
| 数据更新频率 | 日更新 | 蓝海品类数据 |
| 系统可用性 | 99.9% | 月度可用时间 |
| 并发支持 | 1000+ | 同时在线用户 |

### 6.2 安全需求

- 用户密码加密存储（bcrypt）
- JWT Token认证，有效期24小时
- 敏感操作需二次验证
- 数据传输全程HTTPS
- 定期安全漏洞扫描

### 6.3 兼容性需求

- **浏览器**: Chrome/Firefox/Safari/Edge最新版
- **操作系统**: Windows/macOS/iOS/Android
- **响应式**: 桌面端(1200px+)、平板(768px-1199px)、移动端(<768px)

### 6.4 数据需求

- **数据源**: 第三方电商平台公开数据
- **更新频率**: 蓝海数据日更新，趋势数据实时
- **数据保留**: 用户历史数据保留1年
- **备份策略**: 每日全量备份，每周增量备份
